About

I was the Senior UX Designer for Amazon.com's checkout between 2007 and 2011. This page lets customers create, validate, and choose from payment methods. On this project, we got a lot right, including where the careful addition of friction actually increased customer success. But we also learned that listening to customers too closely can lead to wrong decisions.

Amazon.com Checkout · Order Placement Page

  • Reorganize the page to clarify key purchase information
  • Massively reduce pageviews per purchase

About

Every purchase on Amazon.com, except 1-Click orders, happens on this page. It handles a huge number of use cases, including obscure ones we only found during development. This project changed the way I design from mostly Photoshop to almost entirely in HTML and Javascript. I built my own prototyping framework that routed URLs to Javascript templates and loaded data from JSON objects. (These days, I'd use an MVC framework like Backbone.js or AngularJS.)
