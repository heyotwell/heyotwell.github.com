CSS Class Cheatsheet

This page contains many examples of the FrUIT framework. Below is a list of HTML tags and CSS classes along with examples and descriptions of what they do. Implementation examples are on the right. There is a breakdown of grid based alignment contaiers at the bottom of this page.

This page uses no custom styles. Everything you see here is built using universal.css.

List of Available CSS Classes What They Do
.showgrid You can use .showgrid in firebug to inspect alignment. Apply the .showgrid class to the container that holds the whole page to turn the grid background on.
.fl1 .fl2 .fl3, etc .fl stands for "float left." The following number specifies how many columns the container should span. So a container with the class .fl18, for instance, would align to the left side of the screen and be 18 colums wide.
.fr1 .fr2 .fr3, etc .fr stands for "float right." The following number specifies how many columns the container should span. So a container with the class .fr18, for instance, would align to the right side of the screen and be 18 colums wide.
.mr1 .mr2 .mr3, etc .mr stands for "margin right." The following number specifies how many columns the container's margin should span. So a container with the class .mr5, for instance, would push itself off the right side of the browser by 5 columns and use the rest of the available horizontal space.
.ml1 .ml2 .ml3, etc

.ml stands for "margin left." The following number specifies how many columns the container's margin should span. So a container with the class .ml5, for instance, would push itself off the left side of the browser by 5 columns and use the rest of the available horizontal space.

.mrX and .mlX tags can also be combined. Say you want a page to have a 4 column fixed container on the left and a 6 column container fixed on the right. You could then make a center column that expands by applying class=".ml4 .mr6"
.last .last should be applied to the rightmost container classed with .fl and .fr. Containers maintain their alignment by using a 15px right margin. The furthest right container doesn't need this margin.
.fl "Float Left." Containers with this class will float left with no width applied.
.fr "Float Right." Containers with this class will float right with no width applied.
.fw "Full Width." Containers with this class with float left and extend to 100% horizontally.
.clearboth same as style="clear:both" - Forces content to the next empty line

Rationale
Up to this point Amazon has mixed fonts in a very ad-hoc manner. This framework attempts to add some logical sense to this by using Verdana for all fonts 12px and below and Arial for all fonts above 12px. Verdana works well for body-copy and small fonts because it renders well at small sizes and it's letter-spacing is generally more consistent and comfortable to read. At larger sizes, however, Verdana is horsy and extremely wide. Verdana looks very inelegant when bolded at large sizes as well. For this reason we've opted to use Arial for larger fonts and headers.
